# 轻屏壁纸V2.0项目生成计划

## 项目架构
- **前端用户端**：uni-app (Vue 3 + Composition API) - 微信小程序/H5/APP
- **管理员后台**：Vue 3 + Element Plus
- **后端服务**：Cloudflare Wrangler (Workers + D1 + KV)

## 实施步骤

### 1. 创建项目基础目录结构
- 建立完整的项目文件夹结构（admin/、api/、pages/、components/、utils/、wrangler/等）
- 创建必要的空文件夹和基础配置

### 2. 创建uni-app前端用户端核心文件
- **页面组件**：index（首页）、classify（分类）、preview（预览）、my（我的）、search（搜索）、notice（公告）
- **通用组件**：导航栏、壁纸卡片、评分组件等
- **工具函数**：request.js（网络请求）、storage.js（存储）、format.js（格式化）、auth.js（权限）

### 3. 创建管理员后台（Vue 3 + Element Plus）
- **后台页面**：登录、壁纸管理、分类管理、轮播图管理、公告管理
- **后台组件**：表单、表格等通用组件
- **接口封装**：对接Wrangler API

### 4. 创建Wrangler后端配置和代码
- **Workers代码**：路由处理（用户/管理员）、D1数据库操作、KV缓存操作
- **API接口**：用户端接口（轮播图、壁纸、评分等）、管理员接口（登录、增删改查等）

### 5. 创建D1数据库迁移脚本
- 创建数据库表：wallpaper、classify、banner、notice、user_behavior、admin

### 6. 创建公共资源和配置文件
- **样式文件**：uni.scss（全局样式变量）、公共样式
- **配置文件**：package.json、manifest.json、pages.json、wrangler.toml
- **静态资源**：图片资源、tabBar图标

## 技术要点
- 使用Pinia进行状态管理
- 实现JWT权限控制
- 集成Cloudflare CDN加速
- 支持多端兼容（微信小程序/H5/APP）
- 实现缓存策略优化性能
